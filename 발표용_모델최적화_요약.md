# AI 아바타(립싱크 영상) 모델 최적화 — 발표용 요약

> 자세한 기술 기록은 `model_test.md` 참고. 이 문서는 발표/설명용으로 쉽게 풀어쓴 버전입니다.

---

## 1. 우리가 만든 것

사용자가 텍스트로 말을 걸면, AI가 답변을 생성하고 → 그 답변을 음성(TTS)으로 바꾸고 → **입모양이 음성에 맞게 움직이는 아바타 영상**을 실시간으로 만들어 보여주는 시스템입니다.

이 영상 생성을 담당하는 모델이 **MuseTalk**이고, 저희가 가진 GPU는 일반 소비자용 그래픽카드(RTX 4060, 8GB)였습니다.

---

## 2. 첫 번째 도전: "더 빠르게 만들 수 있을까?" (속도 최적화)

### 시도한 것: TensorRT라는 가속 기술

영상 한 번 만드는 데 시간이 오래 걸려서, NVIDIA의 가속 기술(TensorRT)을 적용해 속도를 끌어올리려 했습니다.

### 무슨 일이 있었나

- 속도는 30~100초 → 0.2초로 **엄청나게 빨라짐**
- 그런데 영상에 **색이 깨지거나 픽셀이 노이즈처럼 깨지는 현상**이 발생
- 원인을 추적해보니, 가속 과정에서 일부 연산(LayerNorm)이 정밀도를 버리면서 숫자 계산이 깨지는(NaN) 문제였음
- 여러 방법으로 정밀도를 강제로 맞춰보려 했지만, 이 연산은 NVIDIA 가속 엔진 내부에서 "하나로 묶여서" 손을 댈 수 없는 부분이라는 게 확인됨

### 결론

**TensorRT 가속은 포기**하고, 속도는 느리지만 안정적인 일반 방식(PyTorch + FP16)으로 복귀. 대신 다른 방법(워밍업, 프레임 보간 등)으로 속도를 보완했습니다.

| | TensorRT 시도 | 최종 채택 |
|---|---|---|
| 속도 | 매우 빠름 | 적당히 빠름 (실시간 가능) |
| 화질 문제 | 있음 | 없음 |

## 3. 두 번째 도전: "버퍼링 없이 끊김 없게" (스트리밍 최적화)

영상을 처음부터 끝까지 다 만든 다음 보여주면 너무 느리니, **만드는 즉시 조금씩 전송해서 재생**하는 방식(스트리밍)으로 바꿨습니다. 이 과정에서 여러 문제가 있었습니다.

| 문제 | 원인 | 해결 |
|---|---|---|
| 두 번째 요청부터 느려짐 | Windows가 안 쓰는 GPU 메모리를 자동 회수 | GPU 전원 설정을 "최대 성능"으로 변경 |
| 마지막 부분만 유독 느림 | 처음 보는 영상 크기마다 GPU가 재탐색함 | 서버 시작할 때 모든 크기를 미리 한 번씩 돌려서 준비(워밍업) |
| 재생 시작이 늦음 | 생성 속도가 재생 속도보다 살짝 느림 | 불필요한 대기 제거, 버퍼 기준 조정 |
| 첫 음절이 잘림 | 자동재생이 너무 일찍 시작됨 | 자동재생 제거, 직접 제어 |
| 영상 끝에서 멈춤 | 버퍼 재개 조건이 너무 엄격함 | 조건 완화 |

**결과**: 첫 화면이 뜨기까지 11.8초 → **2~3초**로 단축. 중간에 멈추는 현상도 해결.

## 4. 세 번째 도전: "입모양이 우글우글거려요" (가장 오래 걸린 문제)

### 증상

입 주변이 마치 물결처럼 미세하게 일그러져 보이는 현상. 특히 **배경이 있는 영상**에서 심하게 나타남.

### 원인

AI가 매 프레임(1초에 25번)마다 "얼굴이 어디 있는지"를 **새로 인식**합니다. 이때 인식되는 얼굴 위치가 1~2픽셀씩 미세하게 흔들리는데:

- **단색 배경**에서는 이 흔들림이 거의 안 보임
- **복잡한 배경**(가구, 조명 등)에서는 경계가 흔들릴 때마다 배경이 살짝 보였다 안 보였다 해서 우글거리는 것처럼 보임

### 11가지 시도 (대부분 효과 없었음)

여백 조정, 마스크 부드럽게 처리, 위치 평균내기, 배경 고정시키기, 색상 자연스럽게 합치기(seamless cloning) 등 다양한 방법을 시도했지만 큰 개선은 없었습니다.

### 진짜 원인을 찾다

기존에 쓰던 영상은 **영상 편집 툴로 AI가 배경을 지운 뒤 새 배경을 합성한 영상**이었습니다. 그런데 이 배경 제거 작업도 매 프레임마다 독립적으로 처리되기 때문에, 캐릭터 윤곽선도 같이 흔들립니다.

즉, **두 가지 흔들림(우리 AI의 얼굴 인식 흔들림 + 배경 제거 흔들림)이 겹쳐서** 우글거림이 더 심해진 것이었습니다.

### 최종 해결책

**처음부터 배경이 포함된 형태로 제작된 영상**(3D로 통째로 렌더링된 영상)으로 교체. 이렇게 하니 우글거림이 거의 사라졌습니다.

> 핵심 교훈: 소프트웨어로 고치려 했던 문제가 사실은 **소스 영상 자체의 제작 방식 문제**였음.

## 5. 네 번째 도전: "고개를 움직이면 입이 따로 노는 것 같아요" (떨림 보정)

영상을 바꾸고 우글거림은 해결됐지만, 이번엔 **입 위치가 위아래로 살짝 떨리는** 현상이 남아있었습니다.

### 시도 1: 단순 평균 내기

최근 7개 프레임의 위치를 평균 내서 부드럽게 만듦. 그런데:
- 평균 범위를 넓히면(7→15→23) 떨림은 줄지만, **머리가 실제로 움직일 때 AI가 못 따라가서** 입 위치가 어긋남
- 평균 범위를 줄이면 반대로 떨림이 다시 생김
- 즉, "떨림 제거"와 "움직임 추적"이 서로 trade-off 관계

### 시도 2: 지수 이동 평균(EMA) — 최종 채택

최근 프레임에 더 높은 가중치를 주는 방식으로 변경. 실제 움직임에는 빠르게 반응하면서, 잔잔한 떨림은 걸러내는 균형점을 찾음.
