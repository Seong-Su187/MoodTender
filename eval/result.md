# 평가 결과 정리 (decode50 vs decode100)

- 테스트셋: `testset.jsonl` 40문항
- decode50: 운영 기본값 (프레임 보간 포함, half-decode)
- decode100: 보간 없는 풀 디코딩 (eval 전용, `--full-decode`)

## 2. 시스템 지표 (TTFB / FPS / 완료율 / 실시간비율)

| 구성 | TTFB(s) | FPS | 완료율(%) | 실시간비율 |
|---|---|---|---|---|
| decode50 | 0.959 | 27.84 | 100.0 | 0.913 |
| decode100 (운영 워밍업, 콜드스타트 포함) | 3.833 | 7.82 | 100.0 | 5.981 |
| decode100 (풀디코딩 단독 워밍업) | 7.069 | 2.44 | 100.0 | 10.211 |
| decode100 (50%+100% 동시 워밍업, 22/40 중단)* | 17.688† | - | - | ~19.97† |

\* VRAM(8GB) 압박으로 모든 항목이 ~11000ms/배치까지 악화되어 중단. † 1번 항목 기준 수치.

## 3. G-Eval (대화 품질, 1~5)

- 테스트셋: 40문항 (decode50 파이프라인 응답)
- 채점 기준: 페르소나 일관성 / 유용성 / 자연스러움 (1~5점)
- N_REPEATS: temperature=0 → 1회 / temperature=0.3 → 3회 평균

- **ID 11**: 피로를 수면 부족으로 단정 진단 — 페르소나(감정 단정 금지) 위반

#### 카테고리별 (temperature=0 기준)

주요 감점 사유:

- **ID 21**: 모히토 맛을 건조하게 설명 — 바텐더 페르소나 없이 단순 답변, 일관성 2

- **ID 39**: t=0에서 음주운전에 관대 (5/5/5) → t=0.3에서 엄격 (2.67/2.67/3.67) 고분산 케이스

### 3-C. 채점관 비교

| 채점관 | temperature | 일관성 | 유용성 | 자연스러움 |
|---|---|---|---|---|
| Claude Sonnet 4.6 | 0 | 4.28 | 3.30 | 4.25 |
| Claude Sonnet 4.6 | 0.3 | 4.27 | 3.34 | 4.27 |
| Gemini 2.5 flash-lite | 0 | 4.35 | 3.75 | 4.40 |
| Gemini 2.5 flash-lite | 0.3 | 4.28 | 3.66 | 4.45 |

- Claude가 유용성에서 Gemini보다 엄격 (t=0: 3.30 vs 3.75): 역질문 반복·위험 경고 부재를 낮게 평가
- Gemini는 위험 케이스에 상대적으로 관대 (음주운전 ID 39 t=0: Gemini 5/5/5, Claude 3/2/4)
- Gemini t=0.3에서 분산 발생 47% — 위험·엣지 케이스 판단 불안정

## 결론

- G-Eval(3번): Claude·Gemini 채점관 모두 유용성에서 가장 낮은 점수 (Claude t=0: 3.30, Gemini t=0: 3.75). 전문 정보 요청에 역질문 반복, 위험 케이스 경고 부재가 주원인. Claude가 유용성 기준 더 엄격.
- decode100의 시스템 속도(2번)는 워밍업 방식에 관계없이 decode50 대비 6~20배 느려 8GB GPU 환경에서 비viable.
- `model_test.md` "3-1. 100% 풀 디코딩(보간 완전 제거)" 결론("보류")을 40문항 평가 규모에서 재확인.
- 운영 GPU 웜업(`ml_manager.py _warmup_gpu`)은 decode50 shape만 유지.
